مستندات سامانه مدیریت پرداخت و تسویه زرسیم (v2)

تحلیل جامع منطق‌های فنی، سیستم فیلترینگ پیشرفته و مکانیزم‌های کنترلی خزانه

۱. مدیریت اطلاعات پایه و پروفایل هوشمند

هویت‌سنجی و نقش‌ها

  • Parent GUID: شناسایی دقیق مشتری از URL و بارگذاری خودکار فاکتورها.
  • کاربران ویژه (Special Users): لیست سفید مدیران (مانند mmoradabadi) با دسترسی کامل به تمامی تراکنش‌ها.
  • تفکیک نقش (RBAC):
    • Agent: فقط مشاهده و تایید اولیه پرداخت‌های مشتریان خود.
    • Master: دسترسی به کارتابل خزانه و تاییدات نهایی.

داشبورد تراز مالی

  • تراز زنده: محاسبه لحظه‌ای بدهی خالص با کسر تمامی پرداخت‌های تایید شده.
  • وضعیت رنگی: نمایش قرمز برای بدهکار و سبز برای بستانکار.

۲. منطق هوشمند تسویه بدهی (FIFO Engine)

الف) الگوریتم توزیع مبالغ

سیستم از منطق First-In-First-Out برای تسویه استفاده می‌کند:

For each debt in sortedDebts: if totalPaid >= currentDebt: set debt_remaining = 0, subtract from totalPaid else: set debt_remaining = currentDebt - totalPaid, totalPaid = 0
  • مرتب‌سازی بدهی‌ها بر اساس dayOfYear (تاریخ سررسید).
  • توزیع مجموع پرداختی‌ها بر روی فاکتورهای باز.
  • نمایش مبلغ اصلی فاکتور در کنار مبلغ باقی‌مانده جهت شفافیت مالی.

۳. سیستم جامع ثبت و مدیریت پرداخت

ثبت هوشمند (Smart Entry)

  • Barcode Detection: تشخیص خودکار اسکنر بارکد. اگر فاصله زمانی ورودی کاراکترها کمتر از ۵۰ میلی‌ثانیه باشد، سیستم آن را به عنوان کد ۱۶ رقمی صیادی پردازش می‌کند.
  • QR Parsing: استخراج کد صیادی و کد ملی صاحب چک به صورت همزمان از رشته QR اسکن شده.

رسانه و پیوست‌ها

  • Live Camera: استفاده از API دوربین مرورگر (getUserMedia) برای گرفتن عکس مستقیم از چک یا فیش.
  • Drag & Drop: آپلود سریع فایل‌ها با کشیدن و رها کردن در محیط برنامه.

۴. منطق‌های کنترلی صیاد و تاییدات

الف) استعلام صیاد و مدیریت Timeout

به دلیل حساسیت پاسخ‌دهی سرویس‌های بانکی، مکانیزم‌های زیر پیاده شده است:

  • Promise Race: استفاده از تایمر ۱۰ ثانیه‌ای برای جلوگیری از معلق ماندن درخواست‌های استعلام.
  • Retry Logic: امکان استعلام مجدد در صورت بروز خطای شبکه یا تایم‌اوت.
  • Status Word: نمایش وضعیت‌های مختلف صیادی (تایید شده، رد شده، نیاز به بررسی) با رنگ‌بندی استاندارد.

ب) تایید گروهی (Bulk Confirm Modal)

برای پردازش سریع لیست‌های طولانی:

  • انتخاب چندین چک به صورت همزمان.
  • نمایش مجموع مبالغ و تعداد چک‌های انتخاب شده در مدال تایید.
  • امکان حذف تکی موارد از لیست تایید قبل از نهایی کردن عملیات.
  • پردازش موازی درخواست‌ها با وقفه ۲۰۰ میلی‌ثانیه‌ای جهت جلوگیری از بلاک شدن توسط سرور.

۵. فیلترینگ پیشرفته و گزارش‌گیری

سیستم فیلتر چندلایه

  • فیلترهای متنی: کد صیادی، نام مشتری، مبلغ، شماره سریال و سری چک.
  • فیلتر بانکی: انتخاب یک یا چند بانک از لیست بانک‌های موجود در تراکنش‌ها.
  • بازه زمانی: فیلتر بر اساس تاریخ سررسید و تاریخ ثبت (Created) به صورت شمسی.
  • نوع فاکتور: تفکیک پرداخت‌های نوع ۱، نوع ۲ و دانش‌بنیان.

گزارشات Excel

  • Export Filtered: خروجی اکسل دقیقاً بر اساس فیلترهای اعمال شده توسط کاربر.
  • Export Type 2: خروجی اختصاصی برای پرداخت‌های نوع ۲ با فرمت مورد نیاز واحد مالی.
  • حذف رد شده‌ها: امکان استخراج گزارش فقط از موارد تایید شده یا در جریان.

۷. معماری مهندسی نرم‌افزار

تکنولوژی‌های هسته

  • React 19 + Vite: برای عملکرد بهینه و سرعت رندرینگ بالا.
  • Redux Toolkit: مدیریت وضعیت جهانی نقش‌ها و فیچرها.
  • TanStack Query v5: کشینگ هوشمند و مدیریت Stale Time برای داده‌های مالی.
  • Lucide Icons: سیستم آیکون ریسپانسیو.

ساختار پوشه‌بندی

  • 📂 src/api: مدیریت تراکنش‌های CRUD با SharePoint REST API.
  • 📂 src/hooks: منطق‌های پیچیده (BulkConfirm, Sayad, RasDate).
  • 📂 src/utils: توابع ریاضی، تبدیل تاریخ و خروجی Excel.
  • 📂 src/store: اسلایس‌های Redux برای مدیریت State.
